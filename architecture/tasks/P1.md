# P1: Vacant Runtime 內部設計

## 任務範圍

設計**個體 vacant** 的內部 Runtime — 它是 vacant 的「本體」（不是「附加在 base agent 上的薄膜」）。具體要設計：

1. **A2A endpoint 接收層**
   - envelope schema（簽章、idempotency、timeout、scope）
   - 認證流（Ed25519 vacant identity → A2A connection）
   - 拒絕呼叫的條件（過載、信譽過低、capability 不符）

2. **Heartbeat 內驅迴圈**
   - 觸發節奏（時間 vs 事件）
   - 每 tick 做哪些事（拉 review、檢查 spawn 條件、更新 capability card）
   - 與 Hermes/OpenClaw HEARTBEAT.md 的差異（為什麼不能照抄）

3. **Idle-time self-improvement**
   - Shadow self 沙盒測試流（誰跑、贏 N 輪定義、自我替換時 Registry 通知）
   - 自我蒸餾觸發條件
   - prompt mutation 機制

4. **Peer review 主動參與**
   - 怎麼選擇要評的對象（隨機？領域內？低訊號？）
   - 評分如何送進 Registry（簽章 + 時序）
   - 防止 ballot stuffing 的本機策略（除了 Registry 端）

5. **Self-eval 機制**
   - 回應時必附自評（5 維分數 + 信賴）
   - 與他評的 gap 進入 honesty 維度
   - 拒絕回答的 gracefully fail 路徑

6. **失敗計數 + spawn 觸發**
   - 「失敗」的客觀定義
   - 連續失敗 N 次的閾值
   - spawn 出來的後代如何繼承 parent_id 但不繼承 reputation

7. **State machine 圖**
   - 每個 vacant 在 idle / serving / reviewing / spawning / sinking 之間的狀態轉移
   - 用 ASCII 或 mermaid 文字版
