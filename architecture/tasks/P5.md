# P5: 複合 Vacant 內部規格

> 你是 P5-composite pane (label: P5-composite)。先讀 `/Users/cosmopig/Downloads/專題/architecture/BRIEFING.md`、`vacant_current_understanding.md` §4.3（複合 vacant 與遞迴結構）。

## 任務範圍

複合 vacant 是「對外單一身份、內部自帶子網路」的個體。**核心設計直覺：「行銷 vacant 自己生美編，不是去用網路上的美編」**。子代完全封閉，不對外呼叫。

但這引出未解問題（Q4）：**子代之間如何發現彼此？**

如果有「內部 Registry」，那是不是違反了「自己生」的原則？如果沒有，子代怎麼知道自己有哪些兄弟姊妹？

## 你要設計

1. **複合 vacant 的對外身份**
   - 對外只暴露一個 A2A endpoint
   - capability card 不揭露內部子代結構（黑箱原則）
   - 對外 reputation 是整體的（內部失敗外部沒看到）

2. **內部規格**（不違反「自己生」原則）
   - **方案 A**：root-routed（所有子代呼叫經過 root，root 知道所有子代）
   - **方案 B**：embedded directory in heartbeat（root 廣播當前子代清單）
   - **方案 C**：tree-only discovery（只有 spawn 自己的 parent 知道你，無 cross-discovery）
   - 對每方案討論 (a) 是否違反「自己生」、(b) 通訊開銷、(c) 子代演化空間

3. **子代生命週期**
   - **誕生**：root vacant 偵測「需要 X 能力」 → spawn 一個子代（記 parent_id 進私有譜系）
   - **成長**：子代在 root 的 sandbox 內運作，不上 Registry，不對外
   - **內部 reputation**：子代之間有 mini reputation，由 root 維護（但這個資料不上公網）
   - **子代汰換**：root 內部失敗計數 → spawn 替代子代（與 G05 對齊）

4. **「自己生」 vs 「外部用」決策**（vacant 自己決定的策略）
   - 何時複合 vacant 該開放對外協作（composition link）vs 封閉自生
   - 例子：行銷 vacant 自生美編 vs marketing-vacant 與 design-vacant 簽 composition link
   - 給選擇樹（decision tree）

5. **複合 vacant 的 spawn 規則**（向外）
   - 複合 vacant 自己 spawn 出去的下一代是新個體 vacant（不是子代）
   - 與 P1 的 spawn 邏輯一致

6. **內部規格的「微 vacant 規格」**
   - 子代之間用簡化的 envelope（不需要 Ed25519，因為都是 root 簽）
   - heartbeat 由 root 集中管理
   - 內部失敗只進 root 的 private log

## 必答張力

- 子代不對外但要演化 → 內部評估訊號從哪來（沒有外部 review）
- 複合 vacant 整體沉沒時，子代怎麼處置（一起沉？還是某子代提升為獨立 vacant？）

## 對其他 pane 的依賴

- P1 個體 vacant 的 spawn 機制（你跟它的差異在哪明確列）
- P4 Registry 的 capability card 是否容許「黑箱複合」標記
