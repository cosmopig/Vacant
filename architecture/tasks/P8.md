# P8: 視覺化架構圖

> 你是 P8-visual pane (label: P8-visual)。先讀 `/Users/cosmopig/Downloads/專題/architecture/BRIEFING.md` 與 `vacant_current_understanding.md` 全文。

## 任務範圍

產出一份**單一 HTML 檔**，視覺化地呈現完整 Vacant 架構。這是給使用者直接打開瀏覽器看的「主圖」，幫助他理解所有 pane 的設計如何組成一個整體。

## 工具

**請使用 frontend-design skill** — 它會給你高品質設計指引。先 invoke 它。

技術棧建議：
- 純 HTML/CSS/JS（單檔自包含，無外部依賴）
- 可以用 Mermaid.js 或 ELK（透過 CDN 但要 fallback）
- 互動性：hover 顯示元件細節、click 切換不同視角
- 設計風格：高品質、不要 AI 通用感

## 你要呈現的視角

1. **總覽（landing）**
   - 三層分層：A2A/MCP 通訊層 → Vacant 責任層 → 個體 vacant 居民層
   - 「客戶端 vs 個體 vacant」的明確區分
   - Registry 在邊上的位置

2. **個體 vacant 內部**（從 P1 拿）
   - heartbeat / idle-time / serving / reviewing 狀態圖
   - 內部元件方塊圖

3. **Reputation 流動**（從 P3 拿）
   - 五維分數的雷達圖示意
   - 訊號從多源聚合的箭頭

4. **生命週期**（從 vacant_current_understanding §2.2 拿）
   - 誕生 → 本機養成 → 升格上線 → 網路節點期 → 沉沒
   - 視覺化時間軸

5. **複合 vacant 內部**（從 P5 拿）
   - 對外單一身份 + 內部子代封閉網路
   - marketing-vacant 範例

6. **失敗→競爭者誕生**動畫（重要）
   - 一個失敗的 vacant 觸發 spawn → 新 vacant 加入網路 → 接管呼叫流量
   - 這是 Vacant 的招牌，要做得 visually striking

7. **缺口表**（從 BRIEFING §7 拿）
   - G01-G06 對應到哪個元件解決
   - 可點擊跳到對應細節

## 規格

- **viewport**：1440x900 一螢幕能看完總覽，下面 scroll 是各細部視角
- **配色**：Vacant 概念（責任、空缺、未來人）建議冷色 + 一個 accent
- **響應式**：手機也要能看（至少不破版）
- **效能**：載入 < 2s
- **typography**：用一個有設計感的字體（Inter / IBM Plex / Söhne / Noto Sans TC for 中文）
- **不要 emoji 滿天飛**，乾淨克制

如果其他 pane 還沒寫完，**讀他們的 task 檔（`tasks/P*.md`）也行**，能拿到他們將會產出什麼的概念。
