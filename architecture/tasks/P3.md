## 任務範圍

設計 Vacant 五維 reputation 的**數學機制**，要能直接被工程實作且抗 Goodhart。

具體要：

1. **每維的更新公式**
   - 借鑑 CrS Eq.2: `CrS(i)_t = CrS(i)_{t-1} · (1 + η · CSc(i) · r_t)`，但要改成跨 session 持久化
   - 你必須給五個維度（factual / logical / relevance / honesty / adoption）各自的 (a) 更新公式、(b) 衰減函數（時間衰減）、(c) 信賴區間估計（Bayesian 後驗 / Wilson interval / Beta 分布）

2. **聚合算法**（多源訊號 → 五維分數）
   - 訊號類型：caller review / peer review / self-eval gap / ground truth / adoption
   - 同源降權：同 base model peer review 在聚合時權重折扣（避免 echo chamber）
   - Anti-collusion：互評環、ballot stuffing 偵測（用 graph clustering 之類）
   - **無 LLM 在聚合過程中**（純運算）

3. **抗 Goodhart 機制**
   - 五維獨立、不做 cross-talk → 攻擊者不能單點優化
   - 加入「對抗測試」維度（不寫進回應，但隨機抽 vacant 跑紅隊式測試）
   - 「裝可靠 vs 真可靠」的 obfuscation 偵測（行為熵、回應差異性）
   - 引用 Skalse et al. 2022 的不可能定理，明確接受「不可能完全阻止」並設計 graceful degradation

4. **UCB 呼叫決策**（借自 DRF）
   - 公式：`UCB(i) = mean_score(i) + c · sqrt(ln(N) / n_i)` 給新 vacant 機會
   - caller 指定維度權重時的 UCB 變體
   - 與 stake / attestation（P2 給）如何加進選擇分布

5. **Cold start 的數學處理**
   - 新 vacant 的 prior（與 P2 對齊）
   - 第一次 N 個呼叫的 exploration bonus
   - 「足夠樣本」前不公開純量分數（只給「資料不足」標籤）

6. **可解讀的呈現**（Anti-complacency）
   - reputation 不是單一數字，是「五維 + 信賴區間 + 樣本數 + 多元性訊號」
   - UX 上強迫顯示「資料來源 X% 來自單一模型，多元性低」
