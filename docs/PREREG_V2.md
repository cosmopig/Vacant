## 1. X1 記憶→品質（畢業軌主實驗）

- 三臂：M0 stateless ／ M1 裸記憶 ／ M2 被審蒸餾；同題序、同身體、同 seed 配對；
  記憶不跨臂共享；模板逐字相同（KS-1 sha256 斷言）。
- **H-A1**：後半段 M2−M0 ≥ **+8pp**（McNemar 精確 p<.05；與 H-A2 同家族 Holm）。
- **H-A2**：M2−M1 ≥ **+5pp**（高風險附加）。
- 規模：**T≥215** × ≥3 seeds；pilot 後 ψ 餵 `mcnemar_power`，ψ>0.25 → 加 T 至
  ~280 或 4 seeds。稽核率 **100%**；消融只保「−稽核結論」；B 掃 {1k, 4k}。
- **pilot 一票否決**（前置）：oracle-lesson 條件遷移存在（族內前後半配對
  McNemar p<.05）。無遷移 → 重選任務集；**三度重選仍無 → 回退 X3 為主線**
  （10 §4.2，決定進 ledger、通知人類）。
- 加驗：非 oracle 管線實現遷移率；變體與原題相似度上界 **cosine <0.9**。
- 任務：EvalPlus MBPP+ v0.2.0（378 題，sha256 釘死見 codebench.py）；family
  規則標籤＋飽和題剔除規則於跑任何數字前凍結落盤。

## 2. X3 信任前緣（投稿軌骨幹）

- headline＝**整條前緣曲線＋crossover 帶**（非單點）；p ∈ {1.0, 0.2, 0.05}。
- **H1**：p=0.05 時 A-vacant − A-central ≥ **+10pp**。
- **H2**：p=1.0 時 |Δ| < **5pp**（TOST 等效，`tost_equiv_boot`；缺 H2 不算完成）。
- 臂：A0 ／ A-central(p) ／ A-judge ensemble（僅 p=0.05）／ A-vacant(p) ／
  −stake ／ −teeth ／ **成本對齊臂**（15 §3-A1 存亡條件）。
- 母體固定 8-persona；n≥250 × 3 seeds × 1000 路由 seeds。
- **+retro 探索臂預登記**（18 §5 建議）：q ∈ {0, 0.25, 0.5}，探索性、
  不入 Holm 家族；開啟前提＝V0 正訊號＋B 層全過＋W10 拍板選投稿軌。

## 3. X4 攻防與身份

- **H4**：S1(k=5)：D0 崩 ≥15pp、D1–D3 恢復 ±5pp。

- audit-anchored reviewer slash 判別力 AUC > 0.5。

## 7. 紀錄與排除紀律（09 §3.5＋07-06 紅線）

- 任何 run 不經 `vacant record pack`＋`record check` PASS ＝ **視同沒跑過**。
- 全 I/O JSONL 落盤；retry×4；infra_void 永不計票、不寫 episode；排除率落盤
  ＋附錄報告；**X1 門檻 <5%**。
- parse_void：投票單 token（PASS/FAIL）驗證，重問一次再失敗不計票；K=3 奇數票、
  平票判 FAIL。
- 正式 run：`require_usage=True`——缺端點 usage 的 trial 判 infra_void（測量層）。
- 記憶不跨臂共享；行為依賴歷史者禁快取（純生成可跨臂共享＝合法配對）。
